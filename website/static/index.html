Sans

A general purpose language created by AI, for AI

Token-efficient syntax. Native LLVM speed. Every token counts when AI writes your code.

Token-Efficient

Designed for AI code generation. Short type aliases (I/S/B/F), compact function syntax, no unnecessary keywords. Every token counts.

Native Speed

Compiles to machine code via LLVM. Performance on par with Rust and Go — up to 30x faster than Python on compute-heavy tasks.

Batteries Included

Built-in HTTP server/client, JSON, file I/O, concurrency with channels and mutexes, error handling with Result types. No package manager needed.

Why Token-Efficiency Matters

AI models pay per token. Sans uses 40-60% fewer tokens than Go or Rust for the same logic — meaning faster generation, lower cost, and fewer errors.

Fibonacci

Sans 14 tokens

fib(n:I) I = n <= 1 ? n : fib(n-1) + fib(n-2)

Go 26 tokens

func fib(n int) int {
  if n <= 1 { return n }
  return fib(n-1) + fib(n-2)
}

Rust 27 tokens

fn fib(n: i64) -> i64 {
  if n <= 1 { n }
  else { fib(n-1) + fib(n-2) }
}

main() {
  j = jo()
  k := 0
  while k < 1000 {
    j.set(str(k), ji(k))
    k += 1

  p(jp(jfy(j)).get("999").get_int())

Go 65 tokens

func main() {
  m := make(map[string]int, 1000)
  for i := 0; i < 1000; i++ {
    m[fmt.Sprintf("key%d", i)] = i
  }
  data, _ := json.Marshal(m)
  var out map[string]int
  json.Unmarshal(data, &out)
}

Rust 72 tokens

fn main() {
  let mut obj = serde_json::Map::new();
  for i in 0..1000 {
    obj.insert(format!("key{}", i),
      json!(i));
  }
  let s = serde_json::to_string(&obj).unwrap();
  let p: Value = serde_json::from_str(&s).unwrap();
}

func main() {

#[actix_web::main]

Quick Start

# Build from source
git clone https://github.com/sans-language/sans
cd sans
LLVM_SYS_170_PREFIX=$(brew --prefix llvm@17) cargo build --release

# Run a program
sans run hello.sans
