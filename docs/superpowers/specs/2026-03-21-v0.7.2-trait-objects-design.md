## Priority 1: Trait Objects

### Type System

- New type tag: `TY_DYN_TRAIT = 22`
- Constructor: `make_dyn_trait_type(trait_name:I)` — 32-byte struct, trait name at offset 8
- Short syntax: `dyn Trait` in type positions (function params, variable types, array element types)
- i64_compat: YES — heap-allocated 16-byte struct passed as i64 pointer (consistent with Result/Option)
- Type equality: same trait name = equal
- type_to_string: `"dyn TraitName"`

### Runtime Representation (16 bytes, heap-allocated)

```
offset 0: data pointer (ptr to concrete value, type-erased to i64)
offset 8: vtable pointer (ptr to method table)
```

### Coercion: `x as dyn Trait`

- New expression form: `expr as type` parsed as postfix operator
- Typeck: verify source type has `impl Trait` registered
- Codegen: alloc 16, store data ptr at +0, store vtable ptr at +8, return pointer
- Scope GC: track with `SCOPE_TAG_DYN_TRAIT = 7`

### Vtable Generation

- During typeck `impl Trait for Type` processing, build `trait_impls` map:
  - Key: `"ConcreteType::TraitName"`
  - Value: array of `(method_name, mangled_fn_name)` pairs in trait declaration order
- Pass map through IR context to codegen
- Codegen emits per (Type, Trait) pair:
  ```llvm
  @vtable.Point.Valued = private constant [N x ptr] [
    ptr @Point__value,
    ptr @Point__to_string
  ]
  ```
- Method index = position in trait declaration (0-based)

### Method Dispatch on dyn Trait

When `obj.method(args)` where obj is `dyn Trait`:
1. Load data ptr from trait object offset 0
2. Load vtable ptr from trait object offset 8
3. Look up method index from trait definition
4. GEP into vtable by index, load function pointer
5. Call function pointer with (data_ptr, args...)

IR: `IR_DYN_TRAIT_CALL(trait_obj, method_index, args...)`

### Polymorphic Collections

- `array<dyn Valued>` stores trait object pointers (i64)
- Push: `items.push(x as dyn Valued)`
- Access: `items[i].method()` dispatches through vtable
- No special array handling — trait objects are i64 pointers

### Limitations (deferred to post-1.0)

- No trait inheritance (`trait A: B`)
- No default implementations
- No associated types
- No generic trait bounds (`T: Trait`)
- No `dyn Trait` in generic contexts

## Test Fixtures

### trait_object_basic.sans
```sans
trait Valued { value() I }
struct A { x I }
struct B { y I }
impl Valued for A { value() I = self.x }
impl Valued for B { value() I = self.y }

main() I {
  a = A { x: 10 }
  b = B { y: 20 }
  da = a as dyn Valued
  db = b as dyn Valued
  da.value() + db.value() - 30
}
```
Expected: 0

### trait_object_array.sans
```sans
trait Valued { value() I }
struct A { x I }
struct B { y I }
impl Valued for A { value() I = self.x }
impl Valued for B { value() I = self.y }

main() I {
  items = array<dyn Valued>()
  items.push(A { x: 10 } as dyn Valued)
  items.push(B { y: 20 } as dyn Valued)
  sum := 0
  i := 0
  while i < items.len() {
    sum = sum + items[i].value()
    i = i + 1
    0
  }
  sum - 30
}
```
Expected: 0
