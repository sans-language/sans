# Plan 6e: HTTP Client Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add HTTP client support via an opaque `HttpResponse` built-in type backed by a C runtime library using libcurl.

**Architecture:** `HttpResponse` is an opaque pointer type (stored as i64). HTTP operations (GET, POST, response accessors) are implemented in `runtime/http.c` using libcurl's easy API. The compiler pipeline (typeck -> IR -> codegen) recognizes HTTP built-in functions and methods, lowering them to IR instructions that codegen compiles as calls to the C runtime functions.

**Tech Stack:** Rust (compiler), C with libcurl (runtime), LLVM via inkwell 0.8 (codegen), `cc` linker with `-lcurl`

**Spec:** `docs/superpowers/specs/2026-03-13-plan6e-http-design.md`
