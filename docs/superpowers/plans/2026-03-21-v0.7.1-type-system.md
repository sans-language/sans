# v0.7.1 Type System — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Option<T>, generalize Map<K,V>, add distinct types for opaque handles, Result combinators, and trait objects with dynamic dispatch.

**Architecture:** Each feature follows the Sans compiler pipeline: constants (type/IR tags) → typeck (type resolution + checking) → ir (lowering) → codegen (LLVM emission) → runtime (backing functions). Option<T> follows the exact same pattern as the existing Result<T>. The compiler uses manual memory layout (alloc/store64/load64) with all values as i64.

**Tech Stack:** Sans (self-hosted compiler), LLVM 17 (backend)

**Spec:** `docs/superpowers/specs/2026-03-20-v0.7.1-type-system-design.md`

### Task 1: Option Runtime Module

**Files:**
- Create: `runtime/option.sans`
- Modify: `compiler/main.sans:415` (add to runtime_modules)

- [ ] **Step 1: Create runtime/option.sans**

Follow the exact pattern from `runtime/result.sans`. Option is 16 bytes (tag + value).

```sans
// Option<T> runtime — 16-byte layout: [tag:I, value:I]
// tag: 0 = none, 1 = some

sans_option_some(value:I) I {
  o = alloc(16)
  store64(o, 1)
  store64(o + 8, value)
  o
}

sans_option_none() I {
  o = alloc(16)
  store64(o, 0)
  store64(o + 8, 0)
  o
}

sans_option_is_some(o:I) I {
  tag = load64(o)
  if tag == 1 { 1 } else { 0 }
}

sans_option_is_none(o:I) I {
  tag = load64(o)
  if tag == 0 { 1 } else { 0 }
}

sans_option_unwrap(o:I) I {
  tag = load64(o)
  if tag == 1 { load64(o + 8) } else {
    p("error: unwrap called on none\n")
    exit(1)
    0
  }
}

sans_option_unwrap_or(o:I default_val:I) I {
  tag = load64(o)
  if tag == 1 { load64(o + 8) } else { default_val }
}
```
