# v0.8.2 — Server Production-Ready Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the Sans HTTP server suitable for handling real traffic by adding bounded concurrency, multi-read request assembly, timeouts, graceful shutdown, and input validation.

**Architecture:** All changes are in the runtime (`runtime/server.sans`, `runtime/ssl.sans`) with new setter builtins threaded through the compiler pipeline (typeck → constants → IR → codegen). Configuration uses hardcoded globals with optional setter functions. The `serve(port handler)` and `serve_tls(port cert key handler)` APIs remain unchanged.

**Tech Stack:** Sans compiler (self-hosted), LLVM 17, pthreads for mutex

**Spec:** `docs/superpowers/specs/2026-03-24-v0.8.2-server-production-ready-design.md`

### Task 4: Thread pool with bounded workers in server.sans

Add mutex-protected worker counter to `sans_serve`. Move request reading into the worker. Reject with 503 at capacity.

**Files:**
- Modify: `runtime/server.sans` — `sans_serve`, `sans_serve_accept_loop`, `sans_serve_worker`, new helpers

- [ ] **Step 1: Add mutex init helper**

In `runtime/server.sans`, after the setter functions, add:

```sans
sans_init_worker_mutex() I {
  if sans_worker_mutex == 0 {
    sans_worker_mutex = alloc(64)
    pmutex_init(sans_worker_mutex)
  }
  0
}

sans_worker_inc() I {
  pmutex_lock(sans_worker_mutex)
  c = sans_active_workers
  if c >= sans_max_workers {
    pmutex_unlock(sans_worker_mutex)
    0
  } else {
    sans_active_workers = c + 1
    pmutex_unlock(sans_worker_mutex)
    1
  }
}

sans_worker_dec() I {
  pmutex_lock(sans_worker_mutex)
  sans_active_workers = sans_active_workers - 1
  pmutex_unlock(sans_worker_mutex)
  0
}

sans_drain_read_workers() I {
  pmutex_lock(sans_worker_mutex)
  c = sans_active_workers
  pmutex_unlock(sans_worker_mutex)
  c
}
```

```sans
sans_serve_accept_loop(sfd:I handler:I) I {

    ready = spoll(sfd, 1000)
    if ready > 0 {
      cfd = saccept(sfd)
      if cfd >= 0 {
        if sans_worker_inc() == 1 {
          targ = alloc(16)
          store64(targ, handler)
          store64(targ + 8, cfd)
```

- [ ] **Step 1: Add Content-Length parser**

```sans
sans_parse_content_length(buf:I n:I) I {
  // Search for "content-length:" (case-insensitive) in header bytes
  i := 0
  while i + 15 < n {
    // Check for \r\n before the name (or start of headers)
    if i == 0 || (load8(buf + i - 2) == 13 && load8(buf + i - 1) == 10) {
      if sans_cl_match(buf, n, i) == 1 {
        return sans_cl_parse_value(buf, n, i + 15)
      }
    }
    i += 1
    0
  }
  -1
}

sans_cl_match(buf:I n:I pos:I) I {
  if pos + 15 > n { return 0 }
```
