# v0.7.0 Foundations & Bug Fixes — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix four critical bugs, thread source locations through the compiler pipeline, replace crash-on-first-error with multi-error diagnostics, and establish Linux CI.

**Architecture:** The Sans compiler is self-hosted (~11,600 LOC in compiler/, ~6,500 LOC in runtime/) using manual memory layout (alloc/store64/load64). AST nodes are heap-allocated structs accessed by offset. All changes touch the pipeline: lexer → parser → typeck → IR → codegen. Bug fixes are independent of the pipeline changes.

**Tech Stack:** Sans (self-hosted compiler), LLVM 17 (backend), libc (rand, strtod), GitHub Actions (CI)

**Spec:** `docs/superpowers/specs/2026-03-20-v0.7.0-foundations-design.md`

- [ ] **Step 4: Add strtod extern and float detection to json.sans**

In `runtime/json.sans`, find `sans_json_p_make_number` at line 1126. Before it, add a helper to detect floats and parse via strtod.

First, check if `strtod` is already declared as extern anywhere in the runtime. If not, the compiler needs to handle it as a known extern (it handles `atoi`, `malloc`, etc. already — check `compiler/codegen.sans` for the extern declarations list).

Modify `sans_json_p_make_number` (lines 1126-1131) from:
```sans
sans_json_p_make_number(px:I start:I end:I) I {
  n = sans_json_p_atoi(px, start, end)
  v = sans_json_p_node(px, 2)
  store64(v + 8, n)
  v
}
```

To:
```sans
sans_json_p_make_number(px:I start:I end:I) I {
  src = load64(px)
  is_float := 0
  i := start
  while i < end {
    c = load8(src + i)
    if c == 46 { is_float = 1; i = end } else {
      if c == 101 { is_float = 1; i = end } else {
        if c == 69 { is_float = 1; i = end } else {
          i = i + 1; 0
        }
      }
    }
  }
  if is_float == 1 {
    // Extract substring for strtod
    len = end - start
    buf = alloc(len + 1)
    j := 0
    while j < len {
      store8(buf + j, load8(src + start + j))
      j = j + 1; 0
    }
    store8(buf + len, 0)
    // Parse as double using stof (existing builtin, calls strtod internally)
    // stof returns a float (double) value stored as i64 bits
    fval = stof(buf)
    dealloc(buf)
    v = sans_json_p_node(px, 6)
    store64(v + 8, fval)
    v
  } else {
    n = sans_json_p_atoi(px, start, end)
    v = sans_json_p_node(px, 2)
    store64(v + 8, n)
    v
  }
}
```

Note: `stof` is an existing Sans builtin (typeck.sans:1189, ir.sans:656, codegen.sans:3622). It calls libc `strtod` and returns the double value bitcast to i64. This is exactly what we need — no new infrastructure required.

- [ ] **Step 4: Expand emit_closure_aware_call1 to support 0-8 captures**

In `compiler/codegen.sans` at line 930, the function `emit_closure_aware_call1` currently handles 0, 1, and 2 captures with hardcoded branches. Replace the body to use a switch on `ncaps` from 0 to 8.

The key change: instead of branching for ncaps > 1 and then ncaps > 2, generate:
```llvm
switch i64 %ncaps, label %cap_err [
  i64 0, label %cap0
  i64 1, label %cap1
  i64 2, label %cap2
  i64 3, label %cap3
  i64 4, label %cap4
  i64 5, label %cap5
  i64 6, label %cap6
  i64 7, label %cap7
  i64 8, label %cap8
]
```

Each case loads captures 0..N from the closure object (at offsets 2, 3, ..., N+1 from the object base, each i64), then calls the function with (cap0, cap1, ..., capN, arg).

The `cap_err` label should call a panic function.

Apply the same pattern to `emit_closure_aware_call2` (line 1005) and `emit_closure_aware_call3` (line 1063), adjusting for their additional explicit arguments.

This is a large codegen change. The existing pattern at lines 930-1000 shows how to:
- Check if the pointer is a closure (bit 0 set)
- Load the function pointer from offset 0 of the closure object
- Load ncaps from offset 1
- Load captures from offsets 2+

Follow that existing pattern but extend the switch from 2 cases to 8.

### Task 8: Add Source Location to Other AST Nodes

**Files:**
- Modify: `compiler/parser.sans` (make_function, make_struct_def, make_enum_def, etc.)
- Modify: `compiler/typeck.sans`, `compiler/ir.sans`, `compiler/codegen.sans`

**Important:** Unlike expression and statement nodes, these "other" nodes (make_param, make_function, make_struct_def, etc.) do NOT have a tag at offset 0 — they store data directly (e.g., make_param stores `name` at offset 0, make_function stores `name` at offset 0). The `node_line`/`node_col` universal accessor (which reads size from offset 8) does NOT work for these.

**Approach:** Use fixed-offset accessors for these non-tagged nodes. Append line:col as the last two fields, and add per-type accessors:
```sans
fn_line(f:I) I = load64(f + 56)   // make_function: 48 old + 8 line
fn_col(f:I) I = load64(f + 64)    // make_function: 48 old + 16 col
param_line(p:I) I = load64(p + 24) // make_param: 24 old + 0 line (appended)
param_col(p:I) I = load64(p + 32)  // make_param: 24 old + 8 col
```

Do NOT insert a size field at offset 8 for these nodes — that would break all existing field offsets without the benefit of the universal accessor.
