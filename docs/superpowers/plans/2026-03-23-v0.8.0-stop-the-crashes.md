# v0.8.0 — Stop the Crashes: Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate every way Sans can silently crash or segfault — bounds checking, SIGPIPE handling, per-request panic recovery, and fuzz testing.

**Architecture:** Four independent changes: (1) install SIGPIPE signal ignore in server startup, (2) replace silent-fail array/string bounds with runtime errors, (3) replace `exit(1)` in unwrap with `setjmp`/`longjmp` error boundary in server worker loop, (4) build a fuzzer harness for the lexer/parser integrated into CI.

**Tech Stack:** Sans (self-hosted compiler), LLVM 17 codegen, C runtime interop (signal.h, setjmp.h), GitHub Actions CI

**Spec:** `docs/superpowers/specs/2026-03-23-v0.8.x-security-and-adoption-design.md` (v0.8.0 section)

```bash
# Run a test that should compile successfully but fail at runtime with expected error
run_runtime_error_test() {
  local test_file="$1"
  local expected_error="$2"
  local test_name=$(basename "$test_file" .sans)
  local tmp_out="/tmp/sans_test_${test_name}_$$"

  # Compile — should succeed
  $SANS build "$test_file" -o "$tmp_out" 2>/dev/null
  if [ $? -ne 0 ]; then
    echo "FAIL (compile): $test_file — expected compile to succeed"
    FAILURES=$((FAILURES + 1))
    return
  fi

  # Run — should fail with expected error on stderr
  local stderr_out=$("$tmp_out" 2>&1)
  local exit_code=$?

  if [ $exit_code -eq 0 ]; then
    echo "FAIL (runtime): $test_file — expected non-zero exit"
    FAILURES=$((FAILURES + 1))
  elif echo "$stderr_out" | grep -q "$expected_error"; then
    echo "PASS: $test_file"
  else
    echo "FAIL (error msg): $test_file — expected '$expected_error' in output"
    FAILURES=$((FAILURES + 1))
  fi

  rm -f "$tmp_out"
}
```

- [ ] **Step 1: Add bounds check to IR_ARRAY_SET codegen**

The existing IR_ARRAY_GET bounds check code (lines 2576-2582) is the template:
1. Load array length from metadata
2. `icmp slt index, length` (index < length)
3. `icmp sge index, 0` (index >= 0)
4. AND both conditions
5. Branch: in-bounds → do the store; out-of-bounds → call `sans_bounds_fail`

### Task 5: Panic Recovery — setjmp/longjmp Infrastructure

**Files:**
- Modify: `compiler/codegen.sans` (declare setjmp/longjmp externs)
- Modify: `compiler/constants.sans` (add IR constants if needed)
- Modify: `runtime/result.sans:32-40` (replace exit(1) with longjmp)
- Modify: `runtime/option.sans:28-35` (replace exit(1) with longjmp)
- Modify: `runtime/server.sans` (add setjmp error boundary in worker loop)
- Create: `runtime/panic.sans` (panic recovery globals and helpers)
- Create: `tests/fixtures/unwrap_panic_recovery.sans`

This is the most complex task. The approach:
1. Create a global `jmp_buf` and a flag indicating whether panic recovery is active
2. Server worker loop calls `setjmp` before dispatching to handler
3. `sans_unwrap_fail` / `sans_option_unwrap` check the flag — if recovery is active, call `longjmp` instead of `exit(1)`
4. When `longjmp` fires, server catches it and returns 500

- [ ] **Step 1: Create `runtime/panic.sans` with recovery globals**

```sans
// runtime/panic.sans
// Panic recovery for server error boundaries
// jmp_buf is 200 bytes (enough for both Linux and macOS sigjmp_buf)

// WARNING: These globals are NOT thread-safe. If the server spawns threads
// per connection, concurrent panics will corrupt each other's jmp_buf.
// This is a known limitation (see CLAUDE.md: rc_alloc_head/rc_scope_head
// globals are also not thread-safe). Thread-local storage is a future fix.
sans_panic_buf = alloc(200)
sans_panic_active = 0

sans_panic_enable() I {
  sans_panic_active = 1
  0
}

sans_panic_disable() I {
  sans_panic_active = 0
  0
}

sans_panic_is_active() I {
  sans_panic_active
}

sans_panic_get_buf() I {
  sans_panic_buf
}

sans_panic_fire() I {
  longjmp(sans_panic_buf, 1)
  0
}
```

- [ ] **Step 2: Declare setjmp/longjmp externs in codegen**

In `compiler/codegen.sans`, add extern declarations for setjmp/longjmp. Find where other libc externs are declared (search for `declare` patterns near line 367+) and add:

```llvm
declare i32 @_setjmp(ptr)
declare void @longjmp(ptr, i32)
```

**Portability note:** Use `_setjmp` (not `setjmp`) — on macOS, `setjmp` is often a macro that expands to `sigsetjmp`. `_setjmp`/`longjmp` are the raw versions that don't save/restore signal masks, which is what we want here. Test on both macOS and Linux. If `_setjmp` doesn't link on Linux, fall back to `setjmp`.

Also ensure these are callable from Sans runtime code (panic.sans). Add type checking entries in `compiler/typeck.sans` if needed for `setjmp(buf:I) I` and `longjmp(buf:I val:I)` as built-in functions.

- [ ] **Step 5: Add `sans_http_serve` high-level function with error boundary**

**Architecture note:** The current `server.sans` is a library — `sans_http_accept()` returns a request struct and the user dispatches in their own loop. There is no internal "worker loop" to wrap with setjmp. To add panic recovery, we need to provide a higher-level server API that owns the accept-dispatch loop.

Add a new function `sans_http_serve(port:I handler:I)` to `runtime/server.sans` that:
1. Binds and listens on the port
2. Accepts connections in a loop
3. Wraps handler dispatch with setjmp error boundary
4. On longjmp (unwrap panic), returns 500 and continues serving

```sans
sans_http_serve(port:I handler:I) I {
  // Bind and listen (reuse existing sans_http_listen logic)
  server_fd = sans_http_listen(port)
  sans_ignore_sigpipe()

  while 1 {
    req = sans_http_accept(server_fd)
    if req == 0 { continue }

    // Error boundary: setjmp returns 0 on first call, non-zero on longjmp
    sans_panic_enable()
    result = setjmp(sans_panic_get_buf())
    if result == 0 {
      // Normal path: call user handler
      // handler is a function pointer: (request) -> void
      // Sans uses closure-aware indirect calls (see emit_closure_aware_call1
      // in codegen.sans:1091). The implementer needs to determine how to invoke
      // a function pointer from runtime Sans code. Options:
      //   a) If Sans supports calling function-typed variables directly, use handler(req)
      //   b) If not, sans_http_serve may need a new IR op or codegen support
      //      for indirect calls from runtime code
      //   c) Alternatively, use a global callback registration pattern:
      //      sans_http_set_handler(handler) stores it, sans_http_serve calls it
      handler(req)
    } else {
      // Recovery path: handler panicked
      sans_http_respond(req, 500, "Internal Server Error", "text/plain")
    }
    sans_panic_disable()
  }
  0
}
```

The existing `sans_http_accept()` API remains for users who want manual control (no panic recovery). The new `sans_http_serve()` is the recommended API for production servers.

**Note:** The function pointer invocation (`handler(req)`) depends on how Sans handles indirect calls from runtime code. The compiler uses `emit_closure_aware_call1` (codegen.sans:1091) for this in generated code. If calling a function pointer directly from runtime Sans isn't supported, consider a global callback pattern: `sans_http_set_handler(fn)` stores the handler, and `sans_http_serve` retrieves and calls it internally. `sans_http_respond` is the correct API for sending responses (defined at server.sans:448, takes 4 args: req, status, body, content_type).

- [ ] **Step 1: Write the random Sans file generator**

```python
#!/usr/bin/env python3
# tools/fuzz/generators/gen_random_sans.py
# Generates random/malformed .sans files for fuzz testing

import random
import sys
import string

def gen_random_tokens(n):
    """Generate n random tokens mixing valid and invalid Sans syntax."""
    keywords = ['if', 'else', 'while', 'for', 'in', 'fn', 'struct', 'enum',
                'match', 'import', 'return', 'true', 'false', 'main', 'pub']
    types = ['I', 'S', 'B', 'F', 'V', 'Int', 'String', 'Bool', 'Float']
    operators = ['=', ':=', '+=', '-=', '+', '-', '*', '/', '%', '==', '!=',
                 '<', '>', '<=', '>=', '&&', '||', '!', '?', ':', '.', ',',
                 '(', ')', '{', '}', '[', ']', '=>', '->', '|']

    tokens = []
    for _ in range(n):
        r = random.random()
        if r < 0.15:
            tokens.append(random.choice(keywords))
        elif r < 0.25:
            tokens.append(random.choice(types))
        elif r < 0.40:
            tokens.append(random.choice(operators))
        elif r < 0.55:
            tokens.append(str(random.randint(-1000, 1000)))
        elif r < 0.65:
            # Random identifier
            length = random.randint(1, 20)
            tokens.append(''.join(random.choices(string.ascii_lowercase + '_', k=length)))
        elif r < 0.75:
            # Random string literal
            length = random.randint(0, 50)
            content = ''.join(random.choices(string.printable.replace('"', '').replace('\\', ''), k=length))
            tokens.append(f'"{content}"')
        elif r < 0.85:
            tokens.append('\n')
        elif r < 0.90:
            # Null bytes and unicode edge cases
            tokens.append('\x00')
        elif r < 0.95:
            # Very long identifier
            tokens.append('a' * random.randint(100, 10000))
        else:
            # Random bytes
            tokens.append(''.join(chr(random.randint(0, 255)) for _ in range(random.randint(1, 100))))

    return ' '.join(tokens)

def gen_structured_malformed():
    """Generate structurally plausible but malformed Sans programs."""
    templates = [
        # Unclosed braces
        'main() {\n  x = 42\n',
        # Unclosed string
        'main() {\n  s = "hello\n}',
        # Missing function body
        'foo(x:I)\nmain() { foo(1) }',
        # Deeply nested expressions
        'main() {\n  x = ' + '(' * 500 + '1' + ')' * 500 + '\n}',
        # Empty file
        '',
        # Just whitespace
        '   \n\n\t\t\n   ',
        # Binary garbage
        bytes(random.getrandbits(8) for _ in range(200)).decode('latin-1'),
        # Extremely long line
        'main() { x = ' + '1 + ' * 10000 + '1 }',
        # Duplicate mains
        'main() { p("a") }\nmain() { p("b") }',
        # Recursive type reference
        'struct Foo { x: Foo }\nmain() { f = Foo { x: Foo { x: 0 } } }',
    ]
    return random.choice(templates)

if __name__ == '__main__':
    mode = sys.argv[1] if len(sys.argv) > 1 else 'random'
    if mode == 'random':
        print(gen_random_tokens(random.randint(1, 200)))
    elif mode == 'structured':
        print(gen_structured_malformed())
    else:
        print(gen_random_tokens(random.randint(1, 200)))
```

- [ ] **Step 2: Write the fuzz runner script**

```bash
#!/usr/bin/env bash
# tools/fuzz/fuzz_compiler.sh
# Fuzz the Sans compiler with random/malformed input
# Usage: ./tools/fuzz/fuzz_compiler.sh [iterations] [sans-binary]

set -euo pipefail

ITERATIONS=${1:-10000}
SANS_BIN=${2:-./sans}
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
GENERATOR="$SCRIPT_DIR/generators/gen_random_sans.py"
TMPFILE=$(mktemp /tmp/fuzz_XXXXXX.sans)
CRASHES=0
HANGS=0
TOTAL=0
TIMEOUT_SECS=10

cleanup() {
    rm -f "$TMPFILE" "${TMPFILE%.sans}"
}
trap cleanup EXIT

echo "Fuzzing Sans compiler: $ITERATIONS iterations"
echo "Binary: $SANS_BIN"
echo "---"

for i in $(seq 1 "$ITERATIONS"); do
    TOTAL=$((TOTAL + 1))

    # Alternate between random and structured modes
    if (( i % 3 == 0 )); then
        python3 "$GENERATOR" structured > "$TMPFILE" 2>/dev/null
    else
        python3 "$GENERATOR" random > "$TMPFILE" 2>/dev/null
    fi

    # Run compiler with timeout
    set +e
    timeout "$TIMEOUT_SECS" "$SANS_BIN" build "$TMPFILE" > /dev/null 2>&1
    EXIT_CODE=$?
    set -e

    # Check for crashes (signal exits are 128+signal)
    if [ $EXIT_CODE -ge 128 ] && [ $EXIT_CODE -ne 255 ]; then
        SIGNAL=$((EXIT_CODE - 128))
        echo "CRASH at iteration $i: signal $SIGNAL (exit code $EXIT_CODE)"
        cp "$TMPFILE" "$SCRIPT_DIR/crashes/crash_${i}.sans"
        CRASHES=$((CRASHES + 1))
    elif [ $EXIT_CODE -eq 124 ]; then
        echo "HANG at iteration $i: timed out after ${TIMEOUT_SECS}s"
        cp "$TMPFILE" "$SCRIPT_DIR/hangs/hang_${i}.sans"
        HANGS=$((HANGS + 1))
    fi

    # Progress report every 1000 iterations
    if (( i % 1000 == 0 )); then
        echo "Progress: $i/$ITERATIONS (crashes: $CRASHES, hangs: $HANGS)"
    fi

    # Clean up any compiled output
    rm -f "${TMPFILE%.sans}"
done

echo "---"
echo "Results: $TOTAL iterations, $CRASHES crashes, $HANGS hangs"

if [ $CRASHES -gt 0 ] || [ $HANGS -gt 0 ]; then
    echo "FAIL: Found $CRASHES crashes and $HANGS hangs"
    echo "Crash files saved to: $SCRIPT_DIR/crashes/"
    echo "Hang files saved to: $SCRIPT_DIR/hangs/"
    exit 1
else
    echo "PASS: No crashes or hangs found"
    exit 0
fi
```
