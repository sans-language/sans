# Closures with Captures (v0.3.4) Implementation Plan

> **For agentic workers:** Use superpowers:executing-plans to implement this plan.

**Goal:** Add lambda expressions with implicit variable capture to Sans.

**Architecture:** Lambdas are parsed as expressions, lifted to synthetic top-level functions at IR time. Captures become extra leading parameters. Non-capturing lambdas work with existing map/filter via FnRef. Capturing lambdas produce closure structs `{fn_ptr, n_captures, cap0, cap1, ...}` for direct calls. Map/filter with capturing lambdas deferred to v0.3.5.

**Tech Stack:** Rust (compiler), Sans (test fixtures)

**Syntax:**
```sans
add = |x:I| I { x + 10 }        // non-capturing lambda
f = |x:I| I { x + offset }      // captures offset
nums.map(|x:I| I { x * 2 })     // non-capturing, works with map
```

---

### Task 1: Add Pipe token to lexer

**Files:** `crates/sans-lexer/src/lib.rs`, `crates/sans-lexer/src/token.rs`

- Add `Pipe` variant to `TokenKind` enum in token.rs
- In lib.rs, change the `'|'` handling: currently `||` → Or token, single `|` → error. Change to: `||` → Or, single `|` → Pipe token.
- Add lexer test for `|x:I| I { x + 10 }`

### Task 2: Add Lambda AST node and parse it

**Files:** `crates/sans-parser/src/ast.rs`, `crates/sans-parser/src/lib.rs`

Add to Expr enum:
```rust
Lambda {
    params: Vec<Param>,
    return_type: TypeName,
    body: Vec<Stmt>,
    span: Span,
}
```

Parse in `parse_atom()` when `Pipe` token seen:
1. Consume `|`
2. Parse params (name:Type pairs, space-separated) until next `|`
3. Consume `|`
4. Parse return type
5. Parse `{` body `}`

### Task 3: Type-check lambdas with capture detection

**Files:** `crates/sans-typeck/src/lib.rs`

- Type-check lambda body in a child scope (params + parent locals)
- Detect free variables: variables used in body that are in parent scope but not in params
- Return `Type::Fn { params, ret }` (captures don't affect the type signature)

### Task 4: IR lower lambdas — lift to top-level functions

**Files:** `crates/sans-ir/src/lib.rs`

- Detect captures (free variables in lambda body)
- Generate synthetic function name: `__lambda_N`
- Lift lambda body to a new top-level `IrFunction` with captures as extra leading params
- For non-capturing lambdas: emit FnRef to synthetic function (works with existing map/filter)
- For capturing lambdas: emit StructAlloc for closure struct, store fn_ptr + captured values, result is closure pointer

### Task 5: Codegen — handle closure struct for capturing lambdas

**Files:** `crates/sans-codegen/src/lib.rs`

- Closure structs use existing StructAlloc/FieldStore/FieldLoad codegen (no changes needed for allocation)
- For calling a closure (MethodCall or direct Call on a Fn-typed variable): extract fn_ptr from closure struct, read captures, emit LLVM indirect call with captures + explicit args

### Task 6: E2E tests

**Files:** `tests/fixtures/lambda_*.sans`, `crates/sans-driver/tests/e2e.rs`

Test fixtures:
- `lambda_basic.sans`: `|x:I| I { x + 10 }` — non-capturing lambda, direct call
- `lambda_capture.sans`: `|x:I| I { x + offset }` — captures offset from outer scope
- `lambda_map.sans`: `a.map(|x:I| I { x * 2 })` — non-capturing lambda with map
